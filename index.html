Conway's Game of Life

Implemented Conway's Game of Life as a terminal application which allows editing, saving, and loading of patterns.

Conway's game follows four basic rules:
1. Any live cell with fewer than two live neighbors dies, as if by underpopulation.
2. Any live cell with two or three live neighbors lives on to the next generation.
3. Any live cell with more than three live neighbors dies, as if by overpopulation.
4. Any dead cell with exactly three live neighbors becomes a live cell, as if by reproduction.

Conway invented the Game of Life as a thought experiment to demonstrate how complex systems can arise from simple rules. The behavior of the “cells” is complex, with many various mechanisms that emerge from only four rules. You can see below in the examples that there are many different patterns that emerge:


A common pattern that emerges is a "Blinker", when three cells are in a row with no others nearby, they form a stable oscillating pattern. Other oscillating patterns can be found, such as the "Toad" which is larger and made of six cells in two rows, offset by one.

Many more patterns can be found within the game. I enjoy making random games and watching the different mechanisms interact.


However there is an issue in my code which arises when the size of the game is set to values greater than 300 by 300: due to the computation which is performed to update the array from one state to the next the program begins to lag considerably. The above game, which is 600 by 600, took about half an hour to generate.

The issue is that the rules need to be applied for every cell in the array. My program is written in Python, which while it is a wonderful language for rapid development, it has some performance issues. Compiled languages such as C and Golang are much faster. I estimate that the Golang version of the program is about 100x as fast. However unlike Python, Golang lacks a array manipulation library as robust and well documented as Numpy.

I still intend to write a Golang version of the program, but that will take some time as I'm new to the language and the Numpy equivalent I found (Gonum) is not as extensively documented.
